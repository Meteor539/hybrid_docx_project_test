# 阶段性报告1

本阶段主要完成了毕业设计的前期调研、技术路线确定与原型验证工作。首先围绕图像处理、OCR识别与文档版面分析等方向开展学习与资料整理，阅读并梳理不少于15篇中外文献，掌握文档结构化识别、版面检测、文本检测与文本识别等关键技术的常见实现思路，并完成相关外文文献翻译，为后续论文撰写与系统设计提供理论支撑。其次结合高校毕业论文格式审查的实际需求，明确系统总体目标与检查范围，形成“文档渲染生成页面图像—版面分析与文字识别—规则化校验—可视化标注与结果输出”的整体流程，并完成核心模块的初步拆分与接口设计。当前已在本机完成可运行的程序原型：能够将Word文档转换为多页图像，调用现成文档分析与OCR能力获取页面结构与文本信息，基于预设规则对典型问题进行初步检测，并输出问题列表、统计摘要及带高亮标注的页面结果。通过对样本文档的试跑，已初步暴露出处理耗时、环境依赖与跨平台渲染一致性等问题。下一阶段计划在扩展规则覆盖面与前端展示体验的同时，进一步优化处理性能与部署方式，并评估将识别与分析环节迁移到云端加速的可行性，为后续系统完善与测试验证做好准备。



# 阶段性报告2

这个阶段首先做的是项目重构,一开始我先把项目的整体结构梳理了一遍。之前很多逻辑都在同一条线上，后面想加功能会越来越乱。这次我把代码按模块拆开，把docx、ocr这两块块放到统一框架下管理。保留了基于python-docx的检查逻辑，把旧逻辑接进了新框架。之后重点做了界面对接，把新框架接到了GUI按钮上。现在打开程序后，选文档就能直接点“混合检查”，检查结果会显示在界面里。

这个阶段还做了比较多“结果可读性”的优化。最开始混合检查能跑，但输出不够友好，比如只说某个路径有问题，看不出具体错在哪里。因此我调整了输出结构并将其细化，现在会显示对应位置（路径）、对应段落内容、具体问题项等，以方便后续核对。

目前的状态是核心检查可用，界面可操作；但图像检查这边目前还处在接好了入口、规则还要继续打磨的阶段。

下阶段我的大致规划是：第一，继续解决误报问题，让结果更准确；第二，是完善图像那部分规则，让混合检查能在难点场景上发挥作用。

 

# 阶段性报告3

第三阶段主要做了三部分内容。

第一块是继续补图像侧和混合检查相关功能，把图像侧这条线往前推进了许多。像PDF页面分析、页码相关检查、页眉检查、图题表题位置检查、图表不拆页、每章另起一页这些，都在这个阶段逐步接进系统里。

第二块是把混合检查流程实现。这个阶段已经能自动把docx转成pdf，然后直接接图像侧分析。这样整个检查流程顺下来了，docx 侧负责文档结构和格式属性，图像侧负责页面布局和位置相关内容，最后统一输出结果，系统的整体性比之前强了很多。

第三块是GUI和结果展示。这个阶段我花了比较多时间在界面上，把混合检查结果展示出来了。现在不只是输出文字报错，还能在页面预览里标出一部分问题位置，这样更直观，也更能体现项目和图像处理的关系。另外，界面布局、按钮入口、结果分栏、页码切换、缩放查看、问题定位这些内容，也都陆续完成。

到现在为止，已经基本形成了docx 检查、图像侧检查、GUI 展示三部分结合的整体框架。

下一阶段我准备继续做两件事：一是把页面可视化和定位功能再调稳定一些，减少误定位和漏定位；二是继续补还没完全做好的规则，尤其是图像侧和混合检查里稳定性还不够高的部分。这样后面整体完成度会更高一些。
